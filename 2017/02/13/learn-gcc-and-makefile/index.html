gcc 4个步骤

http://www.cnblogs.com/hqu-ye/articles/3655922.html

  1. 预处理:宏替换，分析头文件
    gcc -E test.c -o test.i
  2. 编译:词法分析、语法分析、语义分析、检查语法错误，编译产生汇编文件
    gcc -S test.i -o test.s
  3. 汇编:汇编产生二进制目标文件
    gcc -c test.s -o test.o
  4. 链接:将目标文件和其它附加文件如链接库文件等链接起来，产生可执行文件
    gcc test.o -o test

不同的后缀文件

.h是头文件,.c是C文件,.cc是C++文件,.i预处理后的C文件,.ii预处理后的C++文件,.s汇编文件,.o目标文件,.a库文件

gcc v.s. g++

http://www.cnblogs.com/zhangsir6/articles/2956798.html
http://stackoverflow.com/questions/172587/what-is-the-difference-between-g-and-gcc/172592#172592
g++ is equivalent to gcc -xc++ -lstdc++ -shared-libgcc (the 1st is a compiler option, the 2nd two are linker options). This can be checked by running both with the -v option (it displays the backend toolchain commands being run).
一般来说c程序用gcc，c++程序用g++

  • .c:gcc看做c程序，g++看做c++程序; .cpp:都看做c++程序
  • 对语法要求有区别，C++语法更加严格
  • gcc不能自动和C++程序使用的库联接；可使用gcc -lstdc++

gcc各种参数

http://www.cnblogs.com/xmphoenix/archive/2011/03/21/1989944.html

  • -L/root/mypath 在/root/mypath下查找库文件
  • -lname 链接libname.a库文件
  • -I/root/mypath 在/root/mypath下查找头文件
  • -Wall 显示所有警告信息(warning all)
  • -Werror，它要求GCC将所有的警告当成错误进行处理
  • -pedantic以ANSI/ISO C标准列出的所有警告 当GCC在编译不符合ANSI/ISO C语言标准的源代码时，如果在编译指令中加上了-pedantic选项，那么源程序中使用了扩展语法的地方将产生相应的警告信息。
  • -fpermissive 不提倡。使用宽松的语法。
  • -shared 此选项将尽量使用动态库，所以生成文件比较小，不过需要系统由动态库.
  • -static 此选项将禁止使用动态库，所以，编译出来的东西，一般都非常大，也不必什么动态连接库，就能运行。gcc优先使用动态库
  • -g选项，产生供gdb调试用的可执行文件：gcc -g helloworld.c
  • -O0, O1, O2, O3优化选项
  • -x language filename 设定文件所使用的语言,使后缀名无效如-xc++
  • -fPIC (64位系统必须) 作用于编译阶段，在编译动态库时(.so文件)告诉编译器产生与位置无关代码(Position-IndependentCode)，若未指定-fPIC选项编译.so文件，则在加载动态库时需进行重定向
  • -ansi，支持ISO C89 programs.
  • -E,-S,-c,-o
  • -pthread 通过pthreads库加入对多线程的支持
  • –lmysqlclient 链接mysql库

include <> 和 “”

  • include<> 引用编译器类库路径里面的头文件或者说是系统默认搜索路径下的头文件
  • include”” 先到制定目录下查找，然后再到系统默认路基查找
